Router on a Stick

A router on a stick, also known as an 'one-armed router' is used to connect different VLANs to one another. Because different VLANs have different IP subnets they cannot use a layer-2 switch to talk to one another. Switches do not route IP traffic (which is layer-3), so a layer-3 device is needed for separate VLANs to communicate with one another. A simple example looks like this:

VLAN A and VLAN B want to talk to one another, but they cannot do so directly through just the switch. If a host on VLAN A wants to send something to VLAN B it will see that the destination IP address is not on the local network and will send the packet to the default gateway. When the packet reaches the switch, the switch will attempt to forward this on to the default gateway's MAC address. If a router is present, then the router can serve as the default gateway and send the packet on to VLAN B. If there were no layer-3 device, then the switch wouldn't know where to send the data. Different IP subnets cannot communicate without a layer-3 device (a router) present.

If the above is setup correctly, then a packet from VLAN A will go to the switch, the switch will send it on to the default gateway, which is the mac address of the router (remember, the switch is using layer-2 addressing to forward things). The router then looks in its layer-3 routing table to find the destination, which is in VLAN B. The router sees that to reach VLAN B it has to send the data back to the switch, which then forwards the data on to VLAN B. The router has the IP information of all the VLANs in the network.

To correctly configure a router on a stick, be sure that the interface which goes from the switch to the router is a trunk interface, and that the router has sub-interfaces on its port. One sub-interface per VLAN.

A router on a stick is a nice way to allow communication between multiple VLANs on a single LAN
